# 재무 분석기2 (jaemu-analyzer-web) — PRD

- 문서 버전: 1.1
- 작성일: 2026-07-11
- 상태: 배포 중 (Vercel Production)

## 1. 배경 및 목적

DART(전자공시시스템) Open API로 여러 상장기업의 재무제표를 비교하고 엑셀로
내려받는 웹앱이다. 브라우저에서 바로 쓸 수 있도록 Next.js + Vercel 서버리스
함수로 구성했으며, GitHub 저장소로 관리하고 Vercel에 자동 배포한다.

별도로 Windows용 Tkinter 데스크톱 버전(`재무분석기2/desktop-app`)이 로컬에도
존재하지만, 이 저장소는 그것과 무관하게 독립적으로 동작하는 웹 버전이다.

## 2. 목표 / 비목표

**목표**
- 브라우저에서 업종 → 회사명 순으로 필터링해 최대 5개 회사를 고르고,
  최대 10개년 재무제표 비교 엑셀을 받는다.
- GitHub 저장소로 관리하고 Vercel에 push 시 자동 배포되게 한다.

**비목표**
- 회원가입/로그인, 사용자별 데이터 저장 (무상태 서비스)
- 정확한 실시간 시가총액 랭킹 보장 (아래 8번 리스크 참고 — 근사치)
- 모바일 네이티브 앱

## 3. 사용자 시나리오

1. 업종 키워드(예: "반도체")를 입력하면 해당 업종의 **시가총액 상위 20개**
   상장사가 후보로 뜬다.
2. 회사명 키워드(예: "삼성")를 추가로 입력하면 그 20개 후보 중에서 다시
   좁혀진다.
3. 후보 중 최대 5개를 체크해서 선택한다.
4. 조회 연도 수(최대 10)를 정하고 "분석하고 엑셀 다운로드"를 누르면 요약비교/
   회사별 상세 시트가 담긴 엑셀이 바로 다운로드된다.

## 4. 기능 요구사항

| ID | 요구사항 | 구현 위치 |
|----|----------|-----------|
| F1 | 1차 필터: 업종 키워드로 후보를 좁힌다 | `app/lib/naverIndustry.ts` |
| F2 | 업종 매칭 시 시가총액 상위 20개를 후보로 제시 | 네이버 증권 업종별시세 + 실시간 시세 API |
| F3 | 2차 필터: 회사명 키워드로 후보를 다시 좁힌다 | `app/api/companies/route.ts` |
| F4 | 업종 매칭이 없으면 회사명 단독 검색으로 폴백 | `app/lib/industry.ts` (KRX KIND 상장법인목록) |
| F5 | 최대 5개 회사 선택, 최대 10개년 조회 | `app/page.tsx`, `app/api/analyze/route.ts` |
| F6 | 회사별 재무제표(DART) 조회 및 비율 계산 | `app/lib/dart.ts` |
| F7 | 요약비교/회사별 상세 시트가 담긴 엑셀 생성·다운로드 | `app/lib/excel.ts` |

## 5. 아키텍처

```
[Browser]
   |-- GET /api/companies?industry=..&name=..   (후보 목록, 업종→시가총액 랭킹→회사명 필터)
   |-- POST /api/analyze { companies, yearCount } (DART 조회 → 엑셀 생성 → 다운로드)
   v
[Vercel Next.js 서버리스 함수 (Node.js, 서울 리전 icn1)]
   |-- DART Open API (재무제표)
   |-- 네이버 증권 (업종 분류 + 시가총액 랭킹)
   |-- KRX KIND corpList (회사명 단독 검색 폴백)
```

- 프론트엔드/백엔드 모두 TypeScript, 별도 Python 런타임 없음
- `DART_API_KEY`는 Vercel 프로젝트 환경변수로만 보관 (클라이언트 노출 없음)

## 6. 외부 데이터 소스 및 이유

| 용도 | 소스 | 비고 |
|------|------|------|
| 재무제표 | DART Open API (`fnlttSinglAcnt`) | 표준화된 요약 계정과목이라 XBRL 계정명 편차가 적음 |
| 업종 분류 + 시가총액 랭킹 | 네이버 증권 (`sise_group*`, 실시간 시세 API) | KRX data.krx.co.kr의 시가총액 API는 세션 쿠키 없이 봇 차단(`LOGOUT`)이 있어 서버리스에서 우회하기 어려움 |
| 회사명 단독 검색(업종 미매칭 시) | KRX KIND 상장법인목록 다운로드 | 세션/쿠키 없이 단순 GET으로 안정적으로 동작 |

## 7. 저장소 / 배포

- GitHub: (연결 예정) — push 시 Vercel 자동 배포
- Vercel: 프로젝트 `cpanoh1/jaemu-analyzer-web` 에 이미 연결/배포됨
  (`https://jaemu-analyzer-web.vercel.app`)
- 환경변수: `DART_API_KEY` — Vercel 대시보드에만 저장, 저장소에는 `.env.example`만 포함

## 8. 알려진 리스크 및 한계

| 리스크 | 영향 | 완화 |
|--------|------|------|
| 업종별 시가총액 랭킹이 네이버 증권 데이터에 의존 | 네이버가 페이지 구조를 바꾸면 파싱 실패 | 파싱 실패 시 에러 메시지로 즉시 노출, 재시도 가능한 구조 |
| DART 서버로의 아웃바운드 요청이 Vercel Node fetch에서 간헐적으로 멈추는 현상 발견 | 요청 실패/타임아웃 | `node:https` + IPv4 강제 + 커스텀 User-Agent로 우회 완료 (`app/lib/dart.ts`) |
| DART 재무제표가 없는 연도/기업 존재 | 일부 셀 공백 | 경고 메시지(`X-Warnings`)로 사용자에게 안내 |
| 회사 5개 × 연도 10개 조회 시 서버리스 타임아웃(60초) 근접 가능 | 요청 실패 | `maxDuration=60`, 회사별 병렬 조회로 완화 |

## 9. 성공 지표

- 배포 URL에서 업종 검색 → 회사 선택 → 엑셀 다운로드가 오류 없이 완료된다.
- GitHub 저장소에 `DART_API_KEY` 등 민감정보가 커밋되지 않는다.
